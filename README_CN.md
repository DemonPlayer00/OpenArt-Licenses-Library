# OpenArt 许可证

OpenArt 许可证是一套面向艺术作品的**信任网络许可协议**，旨在对抗各类可能导致艺术作品缺损的政策、平台规则或技术处理，保障艺术家对作品完整性的控制权，并促进未经修正的艺术作品在艺术家之间的自由流动。

## 最高目标

**构建一个由艺术家自主维护的信任网络（Trust Network），使未经修正的艺术作品能够在不依赖外部法律或平台善意的前提下，安全、完整地在网络中流动，同时完整保留艺术家的署名权等精神权利。**

具体而言：

- **对抗缺损**：任何形式的有损压缩、元数据删除、自动水印、格式强制转换等行为，均被视为对作品完整性的侵害。
- **技术自立**：通过加密压缩包、宽松平台、成员公钥体系等技术手段，使艺术品的传播不依赖于任何可能损害作品的中间方。
- **信任优先于规则**：在本许可证定义的信任网络内部，艺术家的共同承诺优先于任何相冲突的法律、政策或平台条款。
- **个人执行**：许可证不寻求外部司法或行政支持，其效力完全依赖信任网络成员的自愿遵守与相互监督。每位艺术家负责自行评估并执行。

## 许可证版本

OpenArt 许可证目前提供两个独立版本，以适应不同的使用场景：

| 版本 | 适用范围 | 典型用途 |
|------|----------|----------|
| **原始版（非商业信任网络版）** | 非营利性艺术作品 | 个人创作分享、非商业性艺术交流、教育用途、非营利展览等 |
| **商业版** | 商业性艺术作品 | 可销售、授权、用于广告、嵌入商业产品、营利性服务等（须另行获得商业授权） |

> 两个版本的核心条款（作品完整性保护、信任网络机制、技术分发要求、署名权等）完全相同，唯一区别在于是否允许商业利用。**原始版严格禁止任何商业用途**，商业版允许商业利用但可能附带额外的授权条件。

## 许可证作用

### 1. 建立信任网络

- 任何接受本许可证并发布作品的艺术家自动成为信任网络成员。
- 成员之间互为授权，无需逐份签署协议即可自由交换未经修正的作品。
- 信任具有传递性：成员从网络中获取的作品，其权利范围一律以本许可证为准。

### 2. 强制保护作品完整性

- **禁止有损压缩**：JPEG 等有损压缩格式即使用于存档也推定为缺损。
- **禁止修改元数据**：EXIF、XMP、IPTC 等信息必须原样保留。
- **禁止自动添加水印或标识**。
- **禁止平台以任何方式限制原始文件的下载或使用专有格式锁定作品**。
- 违反上述义务的成员将被自动中止成员资格。

### 3. 技术分发标准

- **优先使用“宽松平台”**：不对作品进行有损压缩、不加水印、不修改元数据、不限制原始文件下载的平台。
- **加密压缩包作为备选**：当宽松平台不可用时，必须使用 AES‑256 加密的无损压缩包分发，密码通过安全信道单独传送。
- 公开浏览版本可使用低分辨率或有损格式，但不得替代原始作品在网络成员间分发。

### 4. 署名权不受影响

- 使用或分发作品时，必须以合理方式清晰标注原作者姓名（或笔名）、作品名称及许可证名称。
- 加密压缩包的分发信息中必须包含署名信息，且解密前即可获取。

### 5. 优先于外部规则

- 本许可证的条款在信任网络内部具有最高适用效力。
- 任何声称“因法律强制要求而不得不违反许可证”的行为一概不被接受，违反者立即丧失所有授权。
- 遵守本许可证而导致违反当地法律、政策或平台规则的责任，由行为人自行承担。

## 如何使用本许可证

### 对于艺术家（作品发布者）

1. 在您希望保护的作品（图像、源文件等）的元数据、附随文件或分发说明中，加入以下声明：
- 本作品采用 OpenArt 许可证（非商业信任网络版）进行授权。
- 详细信息参见 [许可证全文链接]
2. 同时提供两种版本：
- **未经修正的原作**（原始格式，无有损压缩）
- **公开浏览版本**（可低分辨率或有损压缩，用于预览）
3. 优先将作品上传到**宽松平台**（如自建 IPFS 节点、尊重原文件的云存储）。如必须使用普通平台，请创建加密压缩包，并通过安全渠道分发密码。
4. 您自动成为信任网络成员。您有权从其他成员处获取同样采用本许可证的作品，但必须遵守许可证的所有义务。

### 对于使用者（获取作品者）

1. 您获得该作品的使用权，前提是您也遵守本许可证（包括不进行有损处理、保留署名、不用于商业目的等）。
2. 您可以将该作品分享给其他信任网络成员，但不得将其发布在非宽松平台或用于商业用途。
3. 如果您违反了本许可证的任何条款（例如将作品上传到有损压缩平台），您的所有授权将自动终止，您必须停止使用并销毁已获取的未经修正版本。

## 技术建议

- **加密压缩**：推荐使用 7‑Zip 创建 `.7z` 格式存档，加密算法 AES‑256，压缩级别选择“存储”或“快速”（确保无损）。
- **宽松平台**：自建基于 IPFS 的分发节点、支持原始文件下载的网盘、成员间直接传输等。
- **成员身份验证**：信任网络成员可自愿维护一份共享公钥列表，用于加密密码的验证分发。

## 限制与免责

- **本许可证不授予任何商业用途的权利（原始版）**。商业使用请联系作者获取商业版许可。
- 作品按“现状”提供，作者不承担因使用作品而产生的任何直接或间接责任。
- 许可证不寻求任何外部法律体系的认可或强制执行，其效力完全来自成员间的自愿遵守与监督。
